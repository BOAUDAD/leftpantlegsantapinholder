Dialogs

Behavior

Scrolling

Avoid scrolling content in a dialog where possible. If the dialog does require scrolling, the footer actions always remain visible. If the scrolling dialog contains a list of options, the title also remains visible at all times so that it can be viewed in conjunction with the options. Otherwise, the title scrolls up with the content.

Dismissing

Dialogs can be dismissed by touching/clicking outside of the dialog or by using the system back button (Android). Clicking outside of the dialog is equivalent to choosing the negative or conservative action (ex. Cancel).


Alerts

No title bars


Try it

 

Alerts are interruptive and prevent users from continuing until they make a decision; therefore, they are only used for urgent situations that require user confirmation.

Most alerts don't require a title and can be summed up with a single sentence. The actions on the alert clearly relate to the question or statement, and clearly describe the outcome of the user's decision.

With title bars


Try it

 

Alerts with title bars are used only in high-risk situations, such as potential loss of data. Use titles that clearly describe the action, as opposed to vague titles like "Are you sure?" or "Warning".

Simple dialogs


Try it

Simple dialogs provide additional details about a list item or actions related to the primary task. They are to be used sparingly since they are more interruptive than dropdowns.

Choosing an option commits the option and closes the dialog, while touching outside the dialog or pressing back cancels the action and closes the dialog.

Confirmation dialogs

Non-scrolling content


Try it

 

Confirmation dialogs require users to explicitly confirm their choice before the option is committed. This is only necessary when there is a good reason for the user to pause before confirming; most selections can be made from a dropdown or a simple dialog without requiring the extra confirmation step from the user.

Scrolling content (no options)


Try it

 

Avoid scrolling content within a dialog wherever possible, and instead explore layouts that are optimal for large amounts of content.

If scrolling content cannot be avoided within a confirmation dialog, use the fixed footer so that the actions are visible at all times.

If the dialog does not contain options to select from, the header can scroll with the content.

Layout & Content Guidelines

Do:

  • Use inline expansion (ex. "advanced options") to disclose additional content within a dialog.
  • Use fixed footers for confirmation dialogs with scrolling content.
  • Use full-screen modals for complex functionality on mobile phones.
  • Use questions and actions indicative of their results, such as "Discard draft?" and "Save."
  • Use dropdowns instead of simple dialogs where possible, as they are less disruptive.
  • Stack buttons when they exceed the maximum width, with the affirmative action above and the dismissive action below.

Don't:

  • Open dialogs from within dialogs. The exception is full-screen dialogs, which can open other dialogs without adding unnecessary visual complexity.
  • Allow scrolling content within dialogs unless other layout options are not feasible.
  • Over-use alert titles. They are only necessary for high-risk situations.
  • Use ambiguous content such as "Warning!" or "OK."

Acceptance Criteria

User testing:

  1. Users can easily close dialogs.
  2. The labels on the action buttons help users clearly understand the results of the actions.

Accessibility testing:

  1. The focus is on the dialog when the dialog is open.
  2. Users can tab through all the controls in the dialog.

Tech testing:

  1. Fixed footers are used in confirmation dialogs with scrolling content.
  2. Buttons that exceed the width of the dialog are stacked, with the affirmative action above and the negative dismissive action below.
  3. No secondary dialogs open from the main dialog, unless the main dialog is full-screen.
