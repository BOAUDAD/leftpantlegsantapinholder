Types of errors

Generally, errors within eCommerce can happen at three levels:

  1. Page level
  2. Container level
  3. Form field level

Page-level examples

404 err Network err

Here are two examples of page-level errors that might occur. One is a 404 "Not found" error that brings the user to a real page instead of a frustrating 404 page. The other shows how control interactivity is suspended so that the user doesn't try to submit data when the network goes down.

Prefer non-blocking errors like toasts over modals whenever possible.

Retry

Sometimes, the content of an entire container may fail to load or render. When that happens, show the user that there was an issue in that spot, and give them an opportunity to reload only that resource so that information on other parts of the page remains unaffected. Container errors should follow the same rules as preloaders (take up no more than 6 (large) or 8 (small) centered columns, or the space of the container's content, whichever is smaller).

When possible, attempt to automatically reload the resource, and indicate to the user that their primary action (reloading) will automatically occur in X number of seconds.
