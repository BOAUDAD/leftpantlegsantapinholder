Navigation (E-Comm)





Rationale for Layout

We settled on this simple side navigation for the following reasons:

  • The left side is an expected location for navigation on desktop, but is also easy to use on tablets, which will help the transition to mobile design.
  • The information architecture of the app is shallow and does not require subnavigation.
  • The size and spacing of the navigation items allow for easy touch on mobile devices.
  • Navigation with exposed text labels performed better in usability testing than icon-only navigation.

Layout & Style Guidelines

  • The nav has a fixed width of 92px. This lack of responsiveness prevents it from consuming too much horizontal real estate.
  • Due to their importance over most icons in the content area, the navigation icons are slightly larger at 32px.
  • The selected icon helps users understand where they are within the app, and is represented with the hex code #47bab3.
  • Because it sits above the content, the navigation casts a drop shadow, in line with material design.

Acceptance Criteria

User testing:

  1. Most test participants understand where the nav buttons will take them.

Accessibility testing:

  1. Font characters are hidden from screen readers in all supported browsers.
  2. Text color-to-background contrast ratio conforms to level AA.

Tech testing:

  1. Icons render properly in all supported browsers.
  2. Colors and sizes match design guidelines provided here.
