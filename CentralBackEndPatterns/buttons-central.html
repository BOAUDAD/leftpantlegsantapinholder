Layout

Horizontal placement

Unless the width of the space allotted to buttons is very restricted (as with the right column in the campaign builder, or on mobile devices), the primary button should be placed to the right of the secondary button, with 12px of space between the buttons.

Vertical placement

When the width of the space allotted to buttons is restricted, such as in the builder flows or on mobile devices, stack the buttons vertically. The primary button should be above the secondary button so the user sees it first when scrolling. Allow 22px of vertical space between the buttons.

Use Cases

Primary button

Primary buttons are the most important action on the screen, such as actions like “Save” or “Launch” that enable form completion. There should only be one primary button on a screen, and most will only have a single line of text.

Secondary button

Wherever possible, avoid using secondary buttons. The secondary button is appropriate for less frequently used actions, or actions that we don’t want the user to click accidentally (such as “Cancel”).
