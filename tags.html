Tag usage

Tags will usually exist inside text fields, where they can be added or deleted. Deletion can be caused by the user clicking the actionable portion of the tag, or by pressing a "backspace" equivalent while focused on the text field.

In the event that the tags are displayed in a read-only area (such as a table row), there will be no call-to action that specifically promises to let the user edit them. Instead, any nearby edit CTAs must reflect the entity that the tags are associated with, and not the tags themselves.

Tag styling

Tags offer a different set of interactions than primary or secondary CTAs, so their coloring and size should reflect that difference.

Dev notes

For removable tags, functionality should be similar to what's shown in this jQuery plugin demo.

Acceptance Criteria

User testing:

  1. Tags used according to categorization/tagging needs of the product.
  2. Context of the page makes clear that the tag vocabulary is open or closed.
  3. Users understand if the tag allows them to search for

Tech testing:

  1. Read-only tags cannot be removed by the user (on the page where they are read-only).
  2. Read-only tags can be clicked only if there is a corresponding search associated with clicking that tag.
  3. Removable tags are removable by clicking their "X" mark or by pressing backspace or delete when cursor is in the appropriate location.
  4. Tags longer than XYZ characters are tooltipped according to tooltip rules.
