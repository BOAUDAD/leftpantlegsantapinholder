Floating Action Buttons

Styles & States


add

Default size (56px)

Floating action buttons are used for a primary action on the screen and feature a circled icon floating above the UI. Use the default 56px size for FABs unless it breaks visual continuity with other items on the screen. The icon inside the button is 24px, consistent with other icons throughout the app.

Mini size (44px minimum)

If the default size breaks visual consistency with other elements on the screen, use a smaller size that is more consistent with the rest of the interface. The minimum recommended size is 44px, consistent with other touch zones throughout the application, while the icon can remain at at 24px.

Layout


Distance from edge

 

FABs are at least 16px from the edge on mobile devices.

 

 

On desktop and tablet, FABs are at least 24px from the edge.



FAB placement

Floating action buttons should be attached to structural elements on a screen, such as sheets and toolbars.



Tooltips


add

To help users understand the function of the floating action button, each FAB should have a tooltip on hover or long press. Read more about tooltips here.

Use Case Rules


One per screen

Use only one floating action button per screen in order to maintain its prominence.


Positive actions only

Do not use the floating action button for minor or destructive actions, such as delete or archive.


No overflow menus

Don't put overflow menus in floating action buttons.


Don't block important elements

While the floating action button should be above content, it should not block other important elements.

Transitions

Floating action buttons are meant to be flexible, and we can use animation to add delightful effects and smooth transitions to the primary actions on the screen.


add

Waves

When the FAB simply triggers an action elsewhere on the screen or navigates to a new screen, use the waves animation that is used on other buttons. In this animation, the waves expand outward from where the button was pressed.

mode_edit

Morphing

A floating action button can also morph into a toolbar upon press or from a toolbar upon scrolling. The toolbar should contain related actions.

Acceptance Criteria

User testing:

  1. Users understand the function of the floating action buttons.
  2. Tooltip content makes sense to users.

Accessibility testing:

  1. Screen readers read tooltips associated with buttons.
  2. Users can tab to floating action buttons.

Tech testing:

  1. FABs meet the minimum size requirement of 44px and icons are 24px.
  2. FABs have appropriate hover and pressed states.
  3. Tooltips appear upon hover for all floating action buttons.
  4. FABs have the specified distance from the screen's edge.
  5. There is only one FAB per screen.
  6. FABs have positive actions.
  7. FABs do not contain overflow menus.
  8. FABs do not block important content.
