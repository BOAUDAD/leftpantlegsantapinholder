The graphs on this card are gray to indicate that they are not actionable, while the actionable items use the default action color.

Graphs are responsive so they can be easily viewed on all screen sizes.

Use the minimum number of vertical and horizontal lines needed to help the user understand the graphs without cluttering the UI.

The highest value listed on the y axis is determined by the largest y value reported on the active tab's graph, and will therefore change depending on the tab selected. All subsequent y values are equal measures apart. Values on the x axis begin with the start date of the campaign and end with the end date (or today's date, whichever is earlier), and are also divided into equal measures. Values on the x axis stay the same, regardless of the selected tab.

Graphs on Report Screens

Normal state

Report graphs use the background color #009688 to indicate that they are actionable (hovering over a graph shows additional details). Graphs are at 80% opacity so that the lines underneath are still visible for reference.

There are six horizontal lines along the y axis, and the maximum is determined by the largest y value on the graph. All subsequent y values are equal measures apart.

There are nine vertical lines aling the x axis. Values on the x axis begin with the start date of the campaign and end with the current date, and are divided into equal measures.

Graphs are responsive so they can be easily viewed on all screen sizes. If the screen is reduced to a size in which lines on either axies are less than 44 pixels apart, the number of lines should decrease to prevent visual clutter.

Flags indicate points in time when the campaign was edited.


Hover / drag

Hovering over a graph (or dragging on a touch device) on an area where there are no change points shows a flag with the date plus the value associated with the y axis. This provides more detail that cannot be precisely interpreted from simply looking at the graph.


Hovering over a flag

Hovering over a change points shows a flag with the date, the value associated with the y axis, and a description of the edit made to the campaign. It is important for users to see what changes occurred on campaigns, particularly if permissions are shared across multiple users, and also to help explain any campaign metrics.


Report layout

Similar to material design tables, titles are left aligned on a white background. Icons are placed in the upper right corner and aligned horizontally with titles.

Labels along the y axis are aligned with the top of the horizontal lines, while labels along the x axis are left-aligned to the vertical lines.

To reduce UI clutter, there is no label associated with the zero value along the y axis, and there is no label associated with the last value along the x axis (it can be easily inferred from the rest of the labels).


Acceptance Criteria

User testing:

  1. For graphs on cards, users understand how to tab between the graphs.
  2. For graphs on cards, users know how to access full report views.
  3. Users understand the numeric values associated with X and Y axes.
  4. The descriptions of campaign edits that appear on hover make sense to users.

Accessibility testing:

  1. Graphs have titles.
  2. Graphs are delivered in HTML format as opposed to images or PDFs.
  3. Each graph has an associated data table that can be downloaded (as an Excel doc or CSV, for example), which is more accessible by screen readers.
  4. The data table contains the title, the X and Y values, and any descriptions of campaign edits associated with certain dates.
  5. Users can tab through values on the graph to hear X and Y values as well as campaign edit descriptions (if this isn't possible, #3 and #4 are sufficient).

Tech testing:

  1. Graphs respond appropriately to different screen sizes.
  2. Hover/drag states are implemented as shown, and work on desktops as well as touchscreen devices.
  3. Flag indicators appear on dates when edits occurred.
  4. Export links effectively download data tables in Excel or CSV format.
  5. The highest and lowest y values on the graph are visible.
  6. The campaign start and end date (or today's date, whichever is earlier) are visible.
  7. X and Y values are shown at regular intervals.
