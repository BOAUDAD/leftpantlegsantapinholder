Icons

Icon accessibility

By default, Fontello utilizes Private Use Area encoding, which in most cases prevents the mapped character from being spoken by screen readers and confusing users with sight impairments. We also recommend using aria-hidden="true" on the i element to prevent cases where screen readers do read the characters in the PUA. A span containing a hidden text label is also included within the i tag, which will be read by screen readers but invisible to sighted users. The markup looks as follows:

<i aria-hidden="true" class="icon-travel icon-xl"><span class="icon-text">Travel</span></i>

General icons

Most icons throughout the UI are 24px with a color of #9e9e9e, unless they are on components with colored backgrounds (such as buttons or cards), in which case the icons are white. If on a white background, the warning icon uses the hex code #e64a19. No additional classes need to be applied to standard icons.

Acceptance Criteria

User testing:

  1. The meaning of the icons is easily understood by most test participants.

Accessibility testing:

  1. Font characters are hidden from screen readers in all supported browsers.
  2. Hidden text labels make the icon functions clear to users with sight impairments.
  3. Icon color-to-background contrast ratio conforms to level AA.

Tech testing:

  1. Icons render properly in all supported browsers.
  2. An area of space at least 44px wide is dedicated to the icon to allow for easier touch.
  3. Colors and sizes match design guidelines provided here.
