Progress Indicators

Styles

When to use

Progress bars are used when content isn't completely loaded, and the interface may or may not know when it will finish loading. These styles for these are the determinate and indeterminate styles, respectively.

Accent colors should be used to avoid confusion with more interacive elements. When practical, progress indicators should be placed only in main content areas (such as containers of cards), or on any light backgrounds where textual or graphic content will appear.

Container placement

Also, loaders should be placed at the smallest container above the items to be loaded whenever possible. For instance, if a group of accounts or cards hasn't loaded yet, the container in which they reside should have a loader.

Explanatory content

Use phrasing that is generic, but still relevant to the items being loaded inside of the container's title. Optionally, add detail to explain how much time is expected, or any next steps the user should take if the items fail to load in the normal text area.

Loader width

Loaders shouldn't occupy any more space than 6 columns or the default width of the content within a contianer, whichever is smaller. It should obey the same padding rules as any text, graphs, or static images.

Confirming actions

Use a circular preloader inside a button when an action will cause a page load or other network call. This lets the user know that their action is being considered until the results of the action can be shown.
